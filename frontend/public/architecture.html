03

How retrieval actually works

The "RAG" in CreatorRAG. This is the clever bit — finding the right needle without reading the whole haystack every time.

🧩

1 · Chunking

The transcript is cut into ~400-token overlapping pieces (~60-token overlap so context isn't severed at the seam). Each chunk keeps the timestamp of its first and last words — that's how citations point at a moment.

🔢

2 · Embeddings

Each chunk → 768 numbers (a vector) via Gemini. We truncate from the default 3072 dims using MRL — ¼ the storage, near-identical quality for short clips. Vectors are L2-normalized so cosine = dot product.

🎯

3 · Search + filter

Your question is embedded the same way, then Qdrant returns the nearest chunks scoped to this pair's two videos. A score floor (0.3) drops weakly-relevant junk before it ever reaches the model.

♻️

4 · MMR re-rank

We over-fetch 16, then Maximal Marginal Relevance picks the final 6 balancing relevance vs. diversity (λ=0.6) — so the model sees varied evidence, not six paraphrases of the same line. Done locally; free.

Why this beats stuffing the whole transcript into the prompt: it's cheaper (you send ~6 short chunks, not 20 minutes of text), faster, and more accurate (the model isn't distracted by irrelevant material). And it scales — search stays fast whether you have 50 chunks or 50 million.
04

The hard parts

The decisions that actually carry the project — the stuff a reviewer cares about.

🧱

Provider interface + fixture kill-switch

Every source (YouTube, IG-Apify, IG-fixture) implements one fetch() contract. Instagram is flaky, so if Apify fails a cached fixture steps in — the live demo never hard-crashes. The whole pipeline is blind to which provider ran.

🎙️

Transcript fallback chain

Captions → Apify SRT → Groq Whisper on downloaded audio. Caption-less videos used to crash with "needs audio_path"; now they gracefully fall to Whisper. Free path is always tried first.

⚙️

Single-pass graph, not ReAct

A tool-calling agent wastes a whole LLM round-trip deciding "I should search." Since every question needs the transcripts, we retrieve unconditionally — one call, ~half the latency. Still LangGraph, still cited.

🚦

Queue-decoupled ingestion

API stays snappy because slow work is pushed to Redis + an arq worker pool. Throughput scales horizontally (more workers) independent of the web tier — the 1000-creators/day story.

🔑

Key rotation + quota failover

Multiple Gemini keys are used round-robin; on a 429 it fails over to the next key (mid-request, before any token streamed). 3 keys ≈ 3× the free daily cap.

💾

Persistent, bounded memory

Chat history lives in Postgres (survives restarts), but only the last 10 messages are replayed — so context cost is capped no matter how long the conversation gets.

05

Where the money goes (nowhere)

The whole thing is engineered to run on free tiers. These are the deliberate cuts — the cost/scale story.

🔁

Never process a video twice

URL-deduped. Already ingested → reused. $0 scraping, transcription & embeddings on a repeat.

📝

Captions before Whisper

If subtitles exist, skip transcription entirely. Audio→text only as a last resort.

⚡

One AI call, not two

Dropped the agent's "decide" round-trip. Halves the LLM spend per question.

✂️

768-dim embeddings

Truncated from 3072. ~4× less vector storage, faster search, no real quality loss.

🪶

Flash-Lite + short answers

Cheaper model, ~512-token cap. Fewer output tokens = lower cost & quicker replies.

🔑

Key rotation

Several free keys, round-robin + failover. Multiplies the free daily quota.

🧠

Bounded memory

Only last 10 messages replayed. Per-request token cost stays flat.

🎯

Local MMR re-rank

Over-fetch then diversify in Python. Better context, zero extra API calls.

🐳

Self-hosted stores

Qdrant + Postgres + Redis in Docker on one droplet. No managed-DB bills.

Cheat-sheet

One row per service, in plain words. Tape this to your monitor.

APIFYScrapes each video — stats, thumbnail, sometimes subtitles. Runs on their IPs so it isn't blocked. The only paid-ish piece (free credits cover the demo).
GROQRuns Whisper: listens to audio and produces the transcript with timestamps. Only when there are no captions. Free & very fast.
GEMINITwo jobs — embeddings (text → meaning-vectors) and the chat answer (Flash-Lite writes the cited reply). Free tier.
QDRANTVector database — stores meaning-vectors, finds the closest matches to your question. Self-hosted, free.
POSTGRESPlain database — metadata, full transcripts, engagement, chat memory. Self-hosted, free.
REDISJob queue — lets slow ingestion run in the background. Also backs the per-IP rate limiter.
LANGGRAPHThe conductor — runs retrieve → generate in order per question. Doesn't do AI itself; it coordinates Gemini + Qdrant.
FASTAPIThe backend brain — routes, dedup, engagement math, chunking, streaming, auth + rate limit.

🧪 What's covered

URL canonicalization & dedup, chunking math, engagement formula, MMR/retrieval logic, embedding key-failover, staleness TTL, provider routing, and an end-to-end integration pass. No network needed for the offline suite.

🚀 How it ships

Backend is dockerized (API + worker share one image w/ ffmpeg). Prod compose adds Caddy for automatic HTTPS. Lands on a DigitalOcean droplet; frontend on Vercel. DB ports stay private; only 80/443 are exposed.

🔬 Measure it yourself

Run python -m scripts.eval_rag — it sends one real question and prints the stage-by-stage breakdown (embed / search / LLM, time-to-first-token, throughput). That's where every number on this page came from.
